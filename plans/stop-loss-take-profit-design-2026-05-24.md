## 1. 现状问题

当前系统**没有任何止盈止损**：
- Paper OMS / Shadow OMS：纯靠模型信号换仓
- optimizer_v2：只管换手约束（max_turnover=10%），不管浮亏
- 唯一的保护是 backtest 里的 `drawdown_stop`（组合级），但 champion/shadow 都没启用

**极端情况**：某持仓股连续 3 天跌停（-27%），系统每天只换 10% 权重，要 10 天才能完全出清。

## 2. 机构止盈止损实践调研

### 2.1 百亿私募的做法

百亿量化私募（幻方/九坤/衍复）的止损不是"个股跌 X% 就卖"这么简单。他们的体系分三层：

**层级 1：个股层面**

| 规则 | 阈值 | 理由 |
|------|------|------|
| 硬止损 | 单票浮亏 > 15-20% | 防止单票黑天鹅拖累组合 |
| 涨跌停保护 | 跌停不卖（卖不出），涨停不买（买不到） | 物理限制 |
| 流动性止损 | ADV 突然降到正常的 30% 以下 | 流动性枯竭风险 |
| ST/退市预警 | 变 ST 立刻卖出 | 交易规则改变 |

注意：私募**不做个股止盈**。原因：
- 量化模型的收益分布是统计性的，截断右尾（止盈）会损害整体期望
- 止盈本质上是在惩罚赢家，和"让利润奔跑"矛盾
- 如果模型说这只股票还在 Top100，就应该继续持有

**层级 2：组合层面**

| 规则 | 阈值 | 理由 |
|------|------|------|
| 组合回撤止损 | 从峰值回撤 > 8-15% | 控制最大回撤 |
| 日度亏损限额 | 单日亏损 > 3-5% | 防止单日灾难 |
| 周度亏损限额 | 周亏损 > 5-8% | 连续亏损降仓 |
| 波动率自适应 | 组合波动率 > 目标的 1.5x | 降仓位 |

**层级 3：Regime 层面**

| 规则 | 触发条件 | 动作 |
|------|---------|------|
| 市场系统性风险 | regime=critical | 降仓到 30%，停止买入 |
| 流动性危机 | 全市场跌停 > 50 只 | 暂停所有交易 |
| 极端事件 | VIX > 40 或等价 | 降仓 + 停止小盘 |

### 2.2 学术研究的共识

1. **固定止损（-X%）是最差的策略**
   - Kaminski (2016) "Trend Following with Managed Futures"：固定止损在趋势策略中降低收益
   - 量化选股不是趋势跟踪，但固定止损同样有害：它会系统性地在低点卖出

2. **波动率调整止损更优**
   - 止损阈值 = K × ATR（平均真实波幅）或 K × 个股波动率
   - 高波动股止损宽一点，低波动股紧一点
   - 避免"所有股票同一阈值"导致高波动股频繁止损

3. **组合层止损 > 个股层止损**
   - 个股层止损会破坏模型的统计特性（truncate return distribution）
   - 组合层止损保护整体（回撤超限时降仓位），不干预个股持有

4. **信号驱动 > 价格驱动**
   - 私募更倾向"模型信号变了就换仓"而不是"价格跌了就止损"
   - 如果 XGB 分数还在 Top100，即使浮亏 -10%，说明模型仍然看好
   - 只有当模型信号 + 价格行为同时恶化才触发卖出

## 3. CC 建议的设计方案

### 3.1 不做的事

- ❌ 不做个股固定止盈（截断右尾损害期望收益）
- ❌ 不做个股固定止损（-10% 就卖 — 太机械，会在底部卖出）
- ❌ 不做日内止损（我们是日频策略，盘中不交易）

### 3.2 要做的事

**Level 1：个股强制退出（只在极端情况触发）**

```python
STOCK_FORCE_EXIT = {
    "max_loss_pct": -0.20,        # 浮亏 > 20% 强制卖出（黑天鹅保护）
    "consecutive_limit_down": 2,   # 连续 2 天跌停 → 下一个能卖的交易日卖出
    "became_st": True,             # 变 ST 立刻卖出
    "suspended_days": 10,          # 停牌 > 10 天 → 复牌首日卖出
}
```

20% 止损阈值的理由：
- XGB Top100 的平均持仓收益率标准差约 3-5%/天
- 20% 约等于 4-6 个标准差，是真正的异常值
- 比私募常见的 15% 宽松一点，因为我们换手率低（10%/天），出清慢

**Level 2：组合层保护**

```python
PORTFOLIO_PROTECTION = {
    "max_drawdown": 0.12,          # 从峰值回撤 > 12% → 降仓到 50%
    "max_daily_loss": 0.04,        # 单日亏损 > 4% → 次日暂停买入
    "max_weekly_loss": 0.07,       # 周亏损 > 7% → 降仓到 60%
    "vol_target": 0.15,            # 年化波动率目标 15%
    "vol_scale": True,             # 实际波动率 > 1.5x 目标时等比例降仓
}
```

**Level 3：Regime 联动**

```python
# 已有 regime controller，加联动：
if regime["alert_level"] == "critical":
    max_position *= 0.3
    disable_new_buys = True
elif regime["alert_level"] == "warning":
    max_position *= 0.6
```

## 4. 请 CX 指导的问题

### Q1：个股止损阈值

-20% 是否合适？还是应该用 ATR 动态调整？

如果用 ATR：
```python
stop_loss = -max(0.15, 3 * stock_atr_20d)  # 至少 -15%，最多 3 倍 ATR
```

### Q2：止损后是否永久禁入

止损卖出的股票应该：
- A：永久不再买入（避免"止损-反弹-再买-再止损"循环）
- B：冷却期 N 天后如果模型信号仍强可以重新买入
- C：不禁入，下次模型选中就买

我倾向 B（冷却 10 个交易日）。

### Q3：组合回撤止损是降仓还是清仓

- 降仓 50%（保留一半仓位，保留反弹机会）
- 清仓（全部变现，等信号恢复再建仓）

我倾向降仓，不清仓。清仓后重建成本很高（100 只股票全部重新买入）。

### Q4：止损应该在 OMS 里还是在 optimizer_v2 里

- OMS 层：直接在执行时检查浮亏，强制覆盖 optimizer 的目标权重
- Optimizer 层：把止损约束加到 optimizer 的 constraints 里

我倾向 OMS 层（更简单直接，止损是硬规则不应该被优化器覆盖）。

### Q5：要不要做 trailing stop（移动止损）

从最高浮盈回撤 X% 时卖出。比如一只股票涨了 +30% 后回撤 -10%（从高点），触发止盈。

私募很少用，因为：
- 破坏模型的统计分布
- 在震荡市频繁触发
- 但对超大盈利仓位有保护价值

我倾向**不做**（至少第一版不做）。

## 5. 实施优先级

```
第一版（立刻做）：
  - 个股 -20% 硬止损
  - 变 ST 立刻卖出
  - 连续跌停保护

第二版（regime 联动后做）：
  - 组合回撤 12% 降仓
  - regime=critical 降仓

第三版（波动率数据稳定后做）：
  - 波动率自适应止损
  - 波动率目标仓位控制
```
