## 1. 背景

所有量价/基本面因子空间已被穷尽（见附录）。LLM 事件因子是唯一发现的正交信号源。

信号链路：
```
东财搜索API → 个股新闻 → MiniMax LLM 结构化提取 → 事件因子 (impact_1d/5d, confidence, novelty)
→ 衰减加权 → 每日因子值 → 叠加到 XGB 预测
```

## 2. 消融结果

### 2.1 子集消融（有事件的股票内部排序）

16 天数据，每天 60-1600 只有事件的股票：

| 指标 | 值 |
|------|-----|
| Avg RankIC | **+0.121** |
| Med RankIC | **+0.128** |
| RICIR | **+1.03** |
| RIC>0 | **88%** (14/16天) |
| 最强单日 | +0.318 (5/11) |
| 最弱单日 | -0.144 (5/13) |
| 样本量 | 1371 stock-days |

**统计显著**：二项检验 P(14/16 正 | 随机) = 0.004。

**对比基准**：XGB 174 模型整体 RankIC = +0.041，Alpha158 最强单因子 +0.082。LLM 事件因子在子集内 +0.121 是最强的。

### 2.2 全A消融（0-fill 无事件股票）

覆盖率 1-3% 时（每天 60-150 只），全A 5000 只上：

| 指标 | 值 |
|------|-----|
| Avg RankIC | **-0.001** (≈ 0) |
| RICIR | -0.10 |

**结论**：覆盖率太低时，4900 只 impact=0 的股票排名随机，淹没了信号。

### 2.3 覆盖率 vs 预估 RankIC

```
覆盖率   预估全A RankIC   XGB叠加后   Excess IR 预估
2%       ~0.000           0.041      2.16 (无变化)
10%      +0.038           0.056      ~2.9
20%      +0.054           0.068      ~3.6

```

公式：`全A_RIC ≈ 子集_RIC × √覆盖率`，`叠加 ≈ √(XGB² + LLM²)`

## 3. 当前实现

### 3.1 数据采集
- **来源**: 东财搜索 API（免费，无需 VPN）
- **并发**: 8 线程采集新闻 + 16 线程 LLM 提取
- **流式写入**: 提取结果实时写盘，timeout 不丢数据
- **自动重试**: 16:30 首次 → 17:30 重试（门槛 news≥1000, events≥500）
- **全A覆盖**: top_n=5000，SIGALRM 120 分钟超时

### 3.2 LLM 提取
- **模型**: MiniMax m2.5-highspeed（¥1/百万 token，每天约 ¥2.5）
- **输出**: event_type, impact_1d, impact_5d, confidence, relevance, novelty, summary
- **18 类事件**: earnings_positive/negative, management_change, product_launch, analyst_upgrade 等
- **质量**: confidence 0.3-0.85，source_quality 按来源加权

### 3.3 因子构建
- 5 个因子：llm_impact_1d_decayed, llm_impact_5d_decayed, llm_event_count, llm_avg_confidence, llm_novelty_score
- 衰减半衰期 5 天 × source_quality 加权
- PIT 安全：只用已发布新闻

### 3.4 覆盖率现状
- 当前每天约 **500-1600 只**有事件（取决于采集量）
- 覆盖率天花板约 **30-40%**（很多小票根本没新闻）
- 已做历史回填 4/27 ~ 5/22（18 天），正在提取中

## 4. 疑问请 CX 指导

### Q1: 覆盖率天花板问题

小票（市值 <30 亿）几乎没有新闻。即使采全A 5000 只，每天只有 ~1500-2000 只有新闻。覆盖率天花板约 30-40%。

**可能的补充数据源**：
- 互动易问答（irm_qa_sh/irm_qa_sz）— 几乎全A覆盖，但质量可能低
- 公告原文（交易所公告）— 全A覆盖，但需要龙虾套餐 API
- 雪球/股吧 — API 已 403，需要爬虫
- 百度搜索指数 — JQData 有接口，但试用流量已用完

**CX 建议**：覆盖率 30-40% 够用吗？还是必须想办法补到 60%+？

### Q2: 因子用法

两种方案：
- **方案 A**: 作为 XGB 特征直接加入 174+N 维（需要覆盖率高，否则大部分是 NaN/0）
- **方案 B**: 作为后处理 score adjustment（XGB 出分后，有事件的股票加/减分）

方案 B 不需要高覆盖率（只调整有事件的股票），但参数（alpha 权重）需要调优。

**CX 建议**：哪种更合理？方案 B 是否可以绕过覆盖率限制？

### Q3: LLM 预测质量

MiniMax 是通用大模型，不是金融专用。impact_1d 的预测可能有系统性偏差。例如：
- 所有 earnings_negative 都给 impact_1d = -0.02 ~ -0.08（很模板化）
- confidence 字段可能不真实反映确定性

**CX 建议**：是否需要做 LLM 预测的校准（calibration）？或者干脆只用 event_type + novelty，不用 impact 数值？

### Q4: 信号衰减

新闻信息可能在发布后几小时就被市场吸收。我们的 label 是 5 天前瞻收益（Ref($close, -5)），可能已经太慢了。

**CX 建议**：
- 是否应该用 1 天 label 来重新验证？
- 衰减半衰期 5 天是否合理？1 天或 2 天会更好？

### Q5: 历史数据不足

东财搜索 API 只能回溯约 1 个月（4/27 ~ 今天）。没有更长的历史新闻数据来做 rolling gate。

**CX 建议**：18-25 天的消融数据在统计上够吗？还是必须等更长时间？

## 5. 已排除的方向（附录）

所有尝试过的信号层改进：

| 方向 | ΔRankIC | 结论 |
|------|---------|------|
| 量价新因子 (moneyflow/block/event) | -0.001 | 噪声 |

| 融资融券 (margin) | 0 | corr=0.91 with amount_raw |
| **LLM 事件因子** | **+0.121 (子集)** | **唯一正交信号** |
